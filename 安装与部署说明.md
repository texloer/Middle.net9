# 安装与部署说明

## 1. 项目发布方式

当前项目采用以下发布方案：

- 发布方式：`framework-dependent`
- 安装包工具：`Inno Setup`
- 目标平台：`win-x64`
- 目标框架：`.NET 9 Desktop Runtime`
- Web 容器：`WebView2 Runtime`
- **不内置** `WebView2 Runtime` 离线安装包

这意味着目标工控机需要预先安装运行时环境，安装包本身只包含业务程序和前端静态资源。

---

## 2. 工控机安装前提

在目标机器上，请先确认已安装以下组件：

### 必装组件

1. `Microsoft Edge WebView2 Runtime`
2. `.NET 9 Desktop Runtime x64`

### 说明

- 本项目是 `WPF` 桌面程序，因此需要安装 `Desktop Runtime`，不是普通的 `.NET Runtime`。
- 本项目运行前端界面依赖 `WebView2`，如果未安装 `WebView2 Runtime`，程序将无法正常显示页面。
- 由于当前安装包未集成这些运行时，所以需要提前在工控机上手动安装。

---

## 3. 开发机生成安装包

项目根目录已提供以下文件：

- `Middle.iss`：`Inno Setup` 安装脚本
- `build-installer.ps1`：一键发布并生成安装包脚本

### 3.1 前置要求

开发机需要安装：

1. `Visual Studio` / `dotnet SDK`
2. `Node.js`（用于构建前端）
3. `Inno Setup 6`

### 3.2 首次准备

如果前端依赖尚未安装，请先执行：

`cd .\my-ui`

`npm install`

---

## 4. 一键生成安装包

在项目根目录执行：

`powershell -ExecutionPolicy Bypass -File .\build-installer.ps1`

如需指定版本号：

`powershell -ExecutionPolicy Bypass -File .\build-installer.ps1 -Version 1.0.0`

### 执行后会自动完成

1. 执行 `Release Publish`
2. 自动构建 React 前端
3. 输出发布文件到 `publish\`
4. 自动调用 `Inno Setup`
5. 输出安装包到 `installer\`

---

## 5. 在 Visual Studio 中生成安装包

当前项目已将安装包生成流程并入 `Release Publish`。

### 操作步骤

1. 将项目切换到 `Release`
2. 右键项目 `Middle`
3. 选择“发布”
4. 执行发布

发布完成后，会自动执行 `Inno Setup` 生成安装包。

### 输出目录

- 发布目录：`publish\`
- 安装包目录：`installer\`
- 默认安装包文件名：`MiddleSetup.exe`

---

## 6. 安装包内容说明

安装包中包含：

- `Middle.exe`
- 项目运行所需的 `.dll`
- `WebAsset\` 前端静态资源
- `config.ini`（如发布目录中存在）

安装包中**不包含**：

- `WebView2 Runtime` 离线安装包
- `.NET Desktop Runtime` 安装包
- `Node.js`
- `my-ui` 源码
- `node_modules`

---

## 7. 工控机部署步骤

### 方式一：使用安装包部署

1. 在工控机上安装：
   - `WebView2 Runtime`
   - `.NET 9 Desktop Runtime x64`
2. 运行 `MiddleSetup.exe`
3. 按安装向导完成安装
4. 启动桌面快捷方式或开始菜单中的 `Middle`

### 方式二：直接拷贝发布目录

如果不走安装包，也可以直接拷贝 `publish\` 目录到工控机运行。

前提仍然是工控机已安装：

- `WebView2 Runtime`
- `.NET 9 Desktop Runtime x64`

---

## 8. 配置文件说明

项目使用 `config.ini` 保存配置。

### 运行时行为

- 在 UI 的 `Config` 页面修改配置后，会自动写回 `config.ini`
- 手动点击保存后，也会立即写回 `config.ini`

### 发布模式下位置

发布模式运行时，`config.ini` 位于程序运行目录。

例如：

`C:\Program Files\Middle\config.ini`

如果文件不存在，程序会自动创建默认配置文件。

---

## 9. 运行模式说明

### Debug 模式

- 使用 `Vite` 开发服务器
- 需要本机安装 `Node.js`
- 页面从 `http://localhost:5173` 加载
- 适合开发调试

### Release 模式

- 使用本地 `WebAsset` 静态资源
- 不需要 `Node.js`
- 适合工控机正式运行

## 10. 常见问题

### Q1：工控机安装后启动空白或报错？

优先检查：

1. 是否安装了 `WebView2 Runtime`
2. 是否安装了 `.NET 9 Desktop Runtime x64`
3. 安装目录下是否存在 `WebAsset\index.html`

### Q2：为什么工控机不需要安装 `Node.js`？

因为发布模式加载的是已经打包好的前端静态文件，而不是 `Vite` 开发服务器。

### Q3：为什么安装包体积不大？

因为当前方案是：

- `framework-dependent`
- 不内置 `WebView2 Runtime`
- 不内置 `.NET Desktop Runtime`

所以安装包只包含项目本体。

### Q4：如果提示找不到 `ISCC.exe` 怎么办？

说明开发机未安装 `Inno Setup 6`，或者安装路径不是默认路径。

可以：

1. 安装 `Inno Setup 6`
2. 或设置环境变量 `ISCC_PATH`

例如：

`$env:ISCC_PATH="C:\Program Files (x86)\Inno Setup 6\ISCC.exe"`
