# 前端与后端通信指南

## 概述
本项目使用 WPF + WebView2 + React 架构，通过 WebView2 的消息通道实现前后端通信。

## 架构说明
- **前端**：React 应用（运行在 WebView2 中）
- **后端**：WPF + MVVM 架构
- **通信桥接**：WebView2 消息通道（`window.chrome.webview`）

## 一、前端向后端发送消息

### 1. JavaScript 发送函数
在你的 React 项目中创建一个通信工具文件（如 `src/utils/webviewBridge.js`）：

```javascript
// src/utils/webviewBridge.js

/**
 * 向后端发送消息
 * @param {string} action - 操作类型
 * @param {object} data - 附加数据
 */
export function sendMessageToBackend(action, data = {}) {
    const message = {
        action: action,
        timestamp: new Date().toISOString(),
        data: data
    };
    
    // 通过 WebView2 的 postMessage 发送到 C# 后端
    if (window.chrome?.webview) {
        window.chrome.webview.postMessage(JSON.stringify(message));
        console.log('✓ 已发送消息到后端:', message);
    } else {
        console.warn('⚠ WebView2 环境未就绪，无法发送消息');
    }
}

/**
 * 监听来自后端的消息
 * @param {function} callback - 消息处理回调函数
 */
export function listenToBackend(callback) {
    if (window.chrome?.webview) {
        window.chrome.webview.addEventListener('message', (event) => {
            try {
                const message = JSON.parse(event.data);
                callback(message);
            } catch (error) {
                console.error('解析后端消息失败:', error);
            }
        });
    }
}
```

### 2. React 组件中使用

```jsx
// src/components/ControlPanel.jsx
import React, { useEffect, useState } from 'react';
import { sendMessageToBackend, listenToBackend } from '../utils/webviewBridge';

function ControlPanel() {
    const [backendStatus, setBackendStatus] = useState('');

    useEffect(() => {
        // 监听后端消息
        listenToBackend((message) => {
            console.log('收到后端消息:', message);
            setBackendStatus(JSON.stringify(message));
        });
    }, []);

    const handleStart = () => {
        sendMessageToBackend('start', { 
            deviceId: 'Device001',
            temperature: 25.5 
        });
    };

    const handleStop = () => {
        sendMessageToBackend('stop', { 
            deviceId: 'Device001' 
        });
    };

    const handleQuery = () => {
        sendMessageToBackend('query', { 
            type: 'status',
            fields: ['temperature', 'pressure', 'state']
        });
    };

    return (
        <div className="control-panel">
            <h2>设备控制面板</h2>
            <button onClick={handleStart}>🚀 启动设备</button>
            <button onClick={handleStop}>🛑 停止设备</button>
            <button onClick={handleQuery}>📊 查询状态</button>
            
            {backendStatus && (
                <div className="status-display">
                    <h3>后端响应：</h3>
                    <pre>{backendStatus}</pre>
                </div>
            )}
        </div>
    );
}

export default ControlPanel;
```

### 3. TypeScript 版本（如果使用 TS）

```typescript
// src/utils/webviewBridge.ts

interface WebViewMessage {
    action: string;
    timestamp: string;
    data?: any;
}

declare global {
    interface Window {
        chrome?: {
            webview?: {
                postMessage: (message: string) => void;
                addEventListener: (event: string, callback: (event: any) => void) => void;
            };
        };
    }
}

export function sendMessageToBackend(action: string, data: any = {}): void {
    const message: WebViewMessage = {
        action,
        timestamp: new Date().toISOString(),
        data
    };
    
    if (window.chrome?.webview) {
        window.chrome.webview.postMessage(JSON.stringify(message));
    }
}
```

## 二、后端处理流程

后端代码已经配置完成：

1. **MainWindow.xaml.cs**：接收前端消息并转发给 ViewModel
2. **MainViewModel.cs**：处理业务逻辑
3. **数据绑定**：StatusBar 自动显示 ViewModel 状态

### 后端处理消息示例

在 `ViewModels/MainViewModel.cs` 中：

```csharp
public void HandleFrontendMessage(string message)
{
    // 解析 JSON（可以使用 System.Text.Json 或 Newtonsoft.Json）
    using var doc = JsonDocument.Parse(message);
    var root = doc.RootElement;
    
    string action = root.GetProperty("action").GetString();
    
    switch (action)
    {
        case "start":
            var deviceId = root.GetProperty("data").GetProperty("deviceId").GetString();
            StartDevice(deviceId);
            break;
            
        case "stop":
            StopDevice();
            break;
            
        case "query":
            QueryDeviceStatus();
            break;
    }
}
```

## 四、常见问题

### Q1: `window.chrome.webview` 未定义？
**A**: 确保页面已完全加载在 WebView2 中。可以在控制台检查：
```javascript
console.log(window.chrome?.webview ? '✓ WebView2 已就绪' : '✗ 非 WebView2 环境');
```

### Q2: 消息发送成功但后端没反应？
**A**: 检查：
1. `MainWindow.xaml.cs` 中 `WebMessageReceived` 事件是否已注册
2. JSON 格式是否正确
3. 在 `OnFrontendMessage` 方法中添加断点调试

### Q3: 如何从后端向前端发送数据？
**A**: 在 ViewModel 中调用：
```csharp
SendDataToFrontend(new { status = "success", temperature = 25.5 });
```

## 五、消息协议建议

建议使用统一的消息格式：

```json
{
    "action": "操作类型（如 start/stop/query）",
    "timestamp": "2024-01-01T12:00:00Z",
    "data": {
        // 具体业务数据
    }
}
```

后端响应格式：
```json
{
    "success": true,
    "message": "操作成功",
    "data": {
        // 返回数据
    }
}
```
