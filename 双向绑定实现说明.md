# WPF 与 React 双向通信与数据绑定实现说明

## ✅ 架构概述

本项目不仅实现了 WPF 原生的 MVVM（ViewModel 与 XAML 之间的双向数据绑定），还实现了 **React 前端与 C# 后端（WebView2）之间的双向通信**，实现了完整的跨平台打通。后台线程能够安全地修改状态并同步至 WPF UI，同时处理前端的命令。

---

## 📋 工作原理 1：WPF 原生数据绑定（ViewModel ↔ XAML）

### 1. ViewModel 实现 `INotifyPropertyChanged`

通过实现该接口并在属性 Setter 中使用 `Dispatcher`，实现线程安全的数据更新。

```csharp
public class MainViewModel : INotifyPropertyChanged
{
    private string _statusMessage = "系统就绪";
    private string _lastReceivedMessage = "";
    private Dispatcher _uiDispatcher;

    public MainViewModel()
    {
        // 捕获UI线程的Dispatcher
        _uiDispatcher = Dispatcher.CurrentDispatcher;
    }

    public string StatusMessage
    {
        get => _statusMessage;
        set
        {
            // 线程安全的属性更新
            if (_uiDispatcher.CheckAccess())
            {
                _statusMessage = value;
                OnPropertyChanged();
            }
            else
            {
                _uiDispatcher.Invoke(() =>
                {
                    _statusMessage = value;
                    OnPropertyChanged();
                });
            }
        }
    }
    
    // 属性变化通知
    public event PropertyChangedEventHandler? PropertyChanged;
    
    protected virtual void OnPropertyChanged([CallerMemberName] string propertyName = null)
    {
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
    }
}
```

### 2. MainWindow 设置 DataContext
在 `MainWindow.xaml.cs` 中，实例化 ViewModel 并将其赋值给窗口的 `DataContext`：
```csharp
public MainWindow()
{
    InitializeComponent();
    
    // 创建 ViewModel 实例并设置数据上下文
    _viewModel = new MainViewModel();
    this.DataContext = _viewModel;
}
```

## 🔄 工作原理 2：React ↔ C# 双向通信

### 场景 1：前端发消息给后端（React → C#）

**流程**：
1. React 调用 `window.chrome.webview.postMessage(...)`。
2. C# 端 `MainWindow` 触发 `WebMessageReceived` 事件，并将消息传递给 `ViewModel` 的 `HandleFrontendMessage`。

**前端代码**（React）：
```javascript
const handleStart = () => {
    window.chrome.webview.postMessage(JSON.stringify({
        action: 'start',
        data: { deviceId: 'Device001' }
    }));
};
```

**后端接收逻辑**（`MainViewModel.cs`）：
```csharp
public void HandleFrontendMessage(string message)
{
    LastReceivedMessage = message; // 更新到绑定的 UI

    // 根据消息内容执行不同操作
    if (message.Contains("\"action\":\"start\""))
    {
        StatusMessage = "设备启动中...";
        SimulateBackgroundTask("设备启动完成");
    }
}
```

### 场景 2：后端发消息给前端（C# → React）

**流程**：
1. ViewModel 定义一个 `Action<string>`，供 `MainWindow` 注入回调。
2. C# 根据业务逻辑调用该 Action，触发 `MyWebView.CoreWebView2.PostWebMessageAsString(...)`。
3. React 前端监听 `message` 事件来接收并更新组件状态。

**委托绑定的注入**（`MainWindow.xaml.cs`）：
```csharp
_viewModel.SendMessageToFrontend = (msg) =>
{
    Dispatcher.Invoke(() =>
    {
        MyWebView.CoreWebView2.PostWebMessageAsString(msg);
    });
};
```

**后端发送数据**（`MainViewModel.cs`）：
```csharp
public void SendDataToFrontend(object data)
{
    // 将对象序列化为 JSON
    string json = System.Text.Json.JsonSerializer.Serialize(data);
    SendMessageToFrontend?.Invoke(json);
}

// 在后台任务完成时调用
private void SimulateBackgroundTask(string resultMessage)
{
    Task.Run(async () =>
    {
        await Task.Delay(2000); // 模拟耗时操作

        StatusMessage = resultMessage; // 同步给 WPF UI

        // 将结果推送给 React 前端
        SendDataToFrontend(new 
        { 
            success = true, 
            message = resultMessage,
            timestamp = DateTime.Now.ToString("HH:mm:ss")
        });
    });
}
```

**前端接收数据**（React / Vite）：
```javascript
useEffect(() => {
    const handleMessage = (event) => {
        try {
            const data = JSON.parse(event.data);
            console.log("收到 C# 返回的数据:", data);
            // 更新 React State 等逻辑...
        } catch (e) {
            console.error("消息解析失败", e);
        }
    };

    window.chrome?.webview?.addEventListener('message', handleMessage);
    
    return () => {
        window.chrome?.webview?.removeEventListener('message', handleMessage);
    };
}, []);
```

## 🎯 总结与最佳实践

1. ✅ **解耦与线程安全**：WPF 通过 `Dispatcher` 实现了无缝线程切换，无论是后台 Task 还是前端消息触发的 ViewModel 属性修改，都不会抛出"跨线程访问"异常。
2. ✅ **两层双向通信**：
   - **C# 内部**：`XAML/View` ↔ `ViewModel`（通过 `INotifyPropertyChanged`）。
   - **跨端通信**：`WebView2 (C#)` ↔ `React App (JS/TS)`（通过 `postMessage` 与 `WebMessageReceived`）。
3. ✅ **架构扩展**：ViewModel 仅仅是业务逻辑，不直接耦合 WebView 对象，通过 Action 回调将 WebView2 调用反转，便于后续进行单元测试或解耦。
